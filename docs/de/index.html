led.run Dokumentation

Verwandeln Sie jeden Bildschirm in ein professionelles digitales Schild. Keine Apps, keine Konten — nur eine URL.

So funktioniert es

led.run verwandelt URLs in digitale Schilder. Der Pfad ist Ihr Text, der Query-String ist Ihre Konfiguration. Öffnen Sie die URL in einem beliebigen Browser und Ihr Schild ist bereit.

led.run/HELLO?theme=neon&color=ff00ff
Text (URL-Pfad) Konfiguration (Query-Parameter)

Der Ablauf ist einfach: Die URL wird in Text und Parameter zerlegt, ein Theme-Renderer übernimmt den Bildschirm und Ihr Schild wird angezeigt. Alles geschieht clientseitig — keine Serververarbeitung.

URL-Protokoll

https://led.run/[text]?[params]
  • Text — der URL-Pfad nach /. Leerzeichen können als + oder %20 kodiert werden.
  • Parameter — Standard-Query-String. Alle Parameter sind optionale „Präferenzhinweise" — Themes entscheiden, ob sie diese verwenden.

Beispiele

led.run/HELLO                          Einfaches Schild
led.run/DO+NOT+DISTURB?t=do-not-disturb   Theme-Auswahl
led.run/OPEN?t=neon&c=ff2d78&bg=0a0008   Benutzerdefinierte Farben
led.run/WELCOME?mode=flow&speed=120      Laufschrift

Anzeigemodi

Themes unterstützen zwei Anzeigemodi:

Schild-Modus

Statische Vollbildanzeige. Der Text wird automatisch angepasst, um den verfügbaren Platz auszufüllen. Ideal für kurze Nachrichten wie „OPEN", „ON AIR" oder „DO NOT DISTURB".

Lauf-Modus

Laufende Schriftanzeige. Der Text scrollt kontinuierlich über den Bildschirm. Ideal für längere Nachrichten, Ankündigungen oder Ticker.

Automatische Erkennung: Wenn kein mode-Parameter gesetzt ist, verwenden die meisten Themes standardmäßig den Schild-Modus für kurzen Text (≤10 Zeichen) und den Lauf-Modus für längeren Text.

Globale Parameter

Diese Parameter werden von allen Themes erkannt. Jedes Theme stellt eigene Standardwerte bereit.

Parameter Alias Typ Standard Beschreibung
theme t string default Theme-ID (siehe Themes)
mode — string auto sign (statisch) oder flow (scrollend)
color c hex je Theme Textfarbe (Farbformat)
bg — hex je Theme Hintergrundfarbe
speed — number je Theme Scrollgeschwindigkeit in px/s (Lauf-Modus)
direction dir string left Scrollrichtung: left oder right
font — string je Theme Schriftfamilie
scale — number 1 Anzeige-Skalierungsfaktor (0.1–1). Siehe Skalierung
fill — hex je Theme Kartenflächen-Hintergrund (Card-Themes, wenn scale < 1)
wakelock w boolean true Bildschirm eingeschaltet lassen (verhindert Ruhezustand)
cursor cur number — Cursor-Auto-Ausblenden-Verzögerung in Millisekunden
lang l string auto UI-Sprache: en, zh, ja, ko, es, fr, de

Farbformat

Farben werden als Hex-Werte ohne das #-Präfix angegeben:

  • 6-stellig RRGGBB — z.B. ff0000 für Rot
  • 8-stellig AARRGGBB — z.B. 80ff0000 für 50% transparentes Rot
led.run/HELLO?color=00ff41          Grüner Text
led.run/HELLO?bg=1a0033&c=ff00ff   Lila Hintergrund, Magenta Text

Skalierung & Card-Themes

Der scale-Parameter (0.1–1) steuert die Anzeigegröße. Sein Verhalten hängt vom Theme-Typ ab:

Card-Themes

Card-Themes (broadcast, street-sign, wood, do-not-disturb, marquee, dot-matrix) verwenden CSS transform: scale() auf einem inneren Wrapper. Wenn scale < 1, schwebt die Karte zentriert auf einem transparenten Hintergrund. Verwenden Sie den fill-Parameter für die Kartenflächenfarbe und bg für den Seitenhintergrund.

Erweiterte Themes

Alle anderen Themes wenden scale als Schriftgrößen-Multiplikator an: Das Auto-Fit-Ergebnis wird mit dem Skalierungswert multipliziert. Hintergrundeffekte bleiben im Vollbild.

Interaktionen

Sobald ein Schild angezeigt wird, können Sie mit Tastatur, Maus oder Touch interagieren.

Tastatur

  • Space Scrollen pausieren / fortsetzen (Lauf-Modus)
  • F Vollbild umschalten
  • Esc Vollbild verlassen

Maus / Touch

  • Einfachklick / Tippen Pausieren / Fortsetzen
  • Doppelklick / Tippen Vollbild umschalten

Werkzeugleiste

Eine schwebende Werkzeugleiste erscheint am unteren Bildschirmrand (wird automatisch mit dem Cursor ausgeblendet). Sie bietet:

  • Vollbild — Vollbildmodus ein- oder ausschalten
  • Drehen — Anzeige um 90° drehen (nützlich für Hochformat-Schilder auf Querformat-Bildschirmen)
  • Übertragen — auf einen externen Bildschirm via Presentation API übertragen
  • Teilen — aktuelle URL zum Teilen kopieren

Übertragung

Die Übertragungsfunktion nutzt die W3C Presentation API, um Ihr Schild auf einem zweiten Bildschirm anzuzeigen. Derzeit unterstützt in Chrome und Edge.

  • Klicken Sie auf die Übertragungs-Schaltfläche in der Werkzeugleiste, um ein Display auszuwählen
  • Der Empfänger zeigt eine saubere Anzeige ohne Werkzeugleiste oder Steuerelemente
  • Die Verbindung bleibt über Seitennavigationen hinweg bestehen
Die Übertragungs-Schaltfläche erscheint nur in Browsern, die die Presentation API unterstützen.
