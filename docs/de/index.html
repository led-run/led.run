led.run Dokumentation

Verwandle jeden Bildschirm in ein Textschild, einen Lichteffekt, einen Sound-Visualizer, eine Uhranzeige, einen QR-Code oder einen Kamerafilter. Keine Apps, keine Konten — nur eine URL.

So funktioniert es

led.run verwendet URLs zur Erstellung von Anzeigen. Der Pfad und die Query-Parameter konfigurieren alles. Öffne die URL in einem beliebigen Browser und deine Anzeige ist bereit — keine Serververarbeitung, keine Konten nötig.

Textschilder

led.run/HELLO?theme=neon&color=ff00ff
Text (URL-Pfad) Konfiguration (Query-Parameter)

Lichteffekte

led.run/light?t=disco&speed=3
Produkt-Präfix Effekt & Einstellungen

Sound-Visualizer

led.run/sound?t=bars&sensitivity=5
Produkt-Präfix Visualizer & Einstellungen

Uhranzeigen

led.run/time?t=digital&format=12h
Produktpräfix Uhr & Einstellungen

QR-Codes

led.run/qr/HELLO?t=neon&ec=H
Inhalt im Pfad Thema & Einstellungen

Kamera-Effekte

led.run/camera?t=ascii&facing=user
Produktpräfix Effekt & Einstellungen

URL-Protokoll

Sechs Produkte, sechs URL-Muster:

Text:   led.run/HELLO?t=neon              Pfad = Textinhalt
        led.run/text/HELLO?t=neon         Explizites /text/-Präfix
Light:  led.run/light?t=disco             Produkt-Präfix + Parameter
Sound:  led.run/sound?t=bars              Produkt-Präfix + Parameter
Time:   led.run/time?t=digital            Produkt-Präfix + Parameter
QR:     led.run/qr/HELLO?t=neon           Inhalt im Pfad + Parameter
Camera: led.run/camera?t=ascii            Produkt-Präfix + Parameter
  • Text — der URL-Pfad nach / wird zum Anzeigetext. Leerzeichen können als + oder %20 geschrieben werden.
  • QR — der zu kodierende Inhalt folgt dem /qr/-Präfix. URLs im Inhalt sollten URL-kodiert sein.
  • Light, Sound, Time & Camera — verwenden reservierte Pfad-Präfixe. Die gesamte Konfiguration erfolgt über Query-Parameter.
  • Alle Parameter sind optionale „Präferenzhinweise“ — Themes/Effekte/Visualizer/Uhren entscheiden, ob sie diese verwenden.
Reservierte Präfixe: text, light, sound, time, qr, camera sind reserviert. Um das Wort „light“ als Text anzuzeigen, verwende led.run/text/light.

Text — Übersicht

Das Text-Produkt verwandelt jeden URL-Pfad in eine Vollbildanzeige. Es enthält 30 integrierte Themes, jedes für einen bestimmten Stil und Anwendungsfall gestaltet. Themes unterstützen zwei Anzeigemodi: statisches Schild und scrollende Laufschrift.

led.run/HELLO                          Einfaches Schild
led.run/DO+NOT+DISTURB?t=do-not-disturb   Theme-Auswahl
led.run/OPEN?t=neon&c=ff2d78&bg=0a0008   Eigene Farben
led.run/WELCOME?mode=flow&speed=120      Scrollende Laufschrift

Anzeigemodi

Text-Themes unterstützen zwei Anzeigemodi:

Schild-Modus

Statische Vollbildanzeige. Der Text wird automatisch an den verfügbaren Platz angepasst. Ideal für kurze Nachrichten wie „OPEN“, „ON AIR“ oder „DO NOT DISTURB“.

Laufschrift-Modus

Scrollende Laufschrift. Der Text scrollt kontinuierlich über den Bildschirm. Ideal für längere Nachrichten, Ankündigungen oder Ticker.

Automatische Erkennung: Wenn kein mode-Parameter gesetzt ist, verwenden die meisten Themes den Schild-Modus für kurzen Text (≤10 Zeichen) und den Laufschrift-Modus für längeren Text.

Text-Parameter

Diese Parameter werden von Text-Themes erkannt. Jedes Theme hat eigene Standardwerte.

Param Alias Typ Standard Beschreibung
theme t string default Theme-ID (siehe Themes)
mode — string auto sign (statisch) oder flow (scrollend)
color c hex pro Theme Textfarbe (Farbformat)
bg — hex pro Theme Hintergrundfarbe
speed — number pro Theme Scrollgeschwindigkeit in px/s (Laufschrift-Modus)
direction dir string left Scrollrichtung: left oder right
font — string pro Theme Schriftfamilie
scale — number 1 Skalierungsfaktor (Text: 0.1–1, Time/QR/Camera: 0.1–3). Siehe Skalierung
fill — hex pro Theme Kartenflächen-Hintergrund (Card-Themes, wenn scale < 1)
wakelock w boolean true Bildschirm eingeschaltet lassen (verhindert Ruhezustand)
cursor cur number — Cursor-Auto-Hide-Verzögerung in Millisekunden
lang l string auto UI-Sprache: en, zh, ja, ko, es, fr, de

Light — Übersicht

Das Light-Produkt verwandelt jeden Bildschirm in einen Vollbild-Lichteffekt. Nutze es als Taschenlampe, Stroboskop, Stimmungslicht oder Ambiente-Display. 18 integrierte Effekte reichen von praktisch (Solid, SOS) bis atmosphärisch (Aurora-Wellen, Lavalampe).

led.run/light?t=solid              Weiße Taschenlampe
led.run/light?t=disco&speed=5      Schnelle Disco-Lichter
led.run/light?t=candle&warmth=9    Extra warme Kerze
led.run/light?t=aurora-waves       Aurora-Lichtbänder
Hinweis: /light ohne ?t= öffnet die Startseite. Teilen-Links enthalten immer ?t=.

Scale & Card Themes

The scale parameter controls display size. Its behavior depends on the theme type:

Card Themes

Card themes (broadcast, street-sign, wood, do-not-disturb, marquee, dot-matrix) use CSS transform: scale() on an inner wrapper. When scale < 1, the card floats centered on a transparent background. Use the fill param to set the card face color, and bg for the page background.

Extended Themes

All other themes apply scale as a font-size multiplier: the auto-fit result is multiplied by the scale value. Background effects remain fullscreen.

Time Product

Clock themes support scale (0.1–3), position, and padding as layout parameters. When scale ≠ 1 or padding > 0, a wrapper is created inside the container with real dimensions (not CSS transform), so themes render at the correct size. Values < 1 shrink the clock; values > 1 enlarge it beyond the container, with overflow:hidden clipping the excess. Use position to control which portion of the enlarged clock is visible (e.g., top-left anchors the top-left corner). padding adds spacing from container edges.

led.run/time?t=digital&scale=0.5&position=top-left&padding=3
led.run/time?t=digital&scale=2&position=center            — 2× zoom, centered
led.run/time?t=digital&scale=2&position=top-left           — 2× zoom, top-left anchor

QR Product

QR themes support the same wrapper mode as Time: scale (0.1–3), position, padding, and fill. When scale ≠ 1 or padding > 0, a wrapper is created. fill controls the outer container background when the wrapper is active.

led.run/qr/HELLO?scale=0.5&position=center&fill=1a1a2e
led.run/qr/HELLO?scale=2&position=top-left

Camera Product

Camera effects support scale (0.1–3) and position for a picture-in-picture effect when scale < 1. No padding or fill params — simplified version of the wrapper mode.

led.run/camera?t=ascii&scale=0.5&position=bottom-right

Casting

The Cast feature uses the W3C Presentation API to display your content on a secondary screen. Currently supported in Chrome and Edge.

  • Click the Cast button in the toolbar to select a display
  • The receiver shows a clean display without toolbar or controls
  • The connection persists across page navigations
The Cast button only appears in browsers that support the Presentation API.
