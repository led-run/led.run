led.run Documentation

Turn any screen into a text sign, light effect, sound visualizer, or clock display. No apps, no accounts — just a URL.

How It Works

led.run uses URLs to create displays. The path and query string configure everything. Open the URL in any browser and your display is ready — no server processing, no accounts needed.

Text Signs

led.run/HELLO?theme=neon&color=ff00ff
Text (URL path) Configuration (query params)

Light Effects

led.run/light?t=disco&speed=3
Product prefix Effect & settings

Sound Visualizers

led.run/sound?t=bars&sensitivity=5
Product prefix Visualizer & settings

Clock Displays

led.run/time?t=digital&format=12h
Product prefix Clock & settings

URL Protocol

Four products, four URL patterns:

Text:   led.run/HELLO?t=neon              Path = text content
        led.run/text/HELLO?t=neon         Explicit /text/ prefix
Light:  led.run/light?t=disco             Product prefix + params
Sound:  led.run/sound?t=bars              Product prefix + params
Time:   led.run/time?t=digital            Product prefix + params
  • Text — the URL path after / becomes the display text. Spaces can be + or %20.
  • Light, Sound & Time — use reserved path prefixes. All configuration is via query params.
  • All parameters are optional “preference hints” — themes/effects/visualizers/clocks decide whether to use them.
Reserved prefixes: text, light, sound, time are reserved. To display the word “light” as text, use led.run/text/light.

Text — Overview

The Text product turns any URL path into a full-screen display. It includes 20 built-in themes, each designed for a specific vibe and use case. Themes support two display modes: static sign and scrolling flow.

led.run/HELLO                          Simple sign
led.run/DO+NOT+DISTURB?t=do-not-disturb   Theme selection
led.run/OPEN?t=neon&c=ff2d78&bg=0a0008   Custom colors
led.run/WELCOME?mode=flow&speed=120      Scrolling marquee

Display Modes

Text themes support two display modes:

Sign Mode

Static full-screen display. Text is auto-fitted to fill the available space. Ideal for short messages like “OPEN”, “ON AIR”, or “DO NOT DISTURB”.

Flow Mode

Scrolling marquee. Text continuously scrolls across the screen. Ideal for longer messages, announcements, or tickers.

Auto-detection: If no mode param is set, most themes default to sign mode for short text (≤10 characters) and flow mode for longer text.

Text Parameters

These parameters are recognized by text themes. Each theme provides its own defaults.

Param Alias Type Default Description
theme t string default Theme ID (see Themes)
mode — string auto sign (static) or flow (scrolling)
color c hex per theme Text color (color format)
bg — hex per theme Background color
speed — number per theme Scroll speed in px/s (flow mode)
direction dir string left Scroll direction: left or right
font — string per theme Font family name
scale — number 1 Display scale multiplier (Text: 0.1–1, Time: 0.1–3). See Scale
fill — hex per theme Card face background (card themes, when scale < 1)
wakelock w boolean true Keep screen on (prevents sleep)
cursor cur number — Cursor auto-hide delay in milliseconds
lang l string auto UI language: en, zh, ja, ko, es, fr, de

led.run/light?t=solid              White flashlight
led.run/light?t=disco&speed=5      Fast disco lights
led.run/light?t=candle&warmth=9    Extra warm candle
led.run/light?t=aurora-waves       Aurora light bands
Note: /light without ?t= opens the landing page. Share links always include ?t=.

Sound — Overview

The Sound product creates real-time audio visualizations driven by your microphone. 12 built-in visualizers transform sound into mesmerizing visuals, from classic spectrum bars to cinematic bokeh fields.

led.run/sound?t=bars              Spectrum analyzer
led.run/sound?t=ocean&waveCount=10 Ocean with more waves
led.run/sound?t=spikes            Radial spikes
Microphone permission: Sound visualizers require microphone access. Your browser will prompt for permission on first use. Audio is processed locally — nothing is recorded or transmitted.

Time — Overview

The Time product turns your screen into a full-screen clock display. Choose from 14 built-in clock themes, from classic 7-segment LED to nixie tubes and sundials. Each theme renders the current time with its own unique visual style.

led.run/time?t=digital                 Default digital clock
led.run/time?t=nixie&color=ff8833      Nixie tube clock
led.run/time?t=analog&style=modern     Modern analog clock
led.run/time?t=lcd&format=12h          LCD watch, 12-hour
Note: /time without ?t= routes to the landing page. Share links always include ?t=, e.g. led.run/time?t=digital.

Color Format

Colors are specified as hex values without the # prefix:

  • 6-digit RRGGBB — e.g. ff0000 for red
  • 8-digit AARRGGBB — e.g. 80ff0000 for 50% transparent red
led.run/HELLO?color=00ff41          Green text
led.run/HELLO?bg=1a0033&c=ff00ff   Purple bg, magenta text

Scale & Card Themes

The scale parameter controls display size. Its behavior depends on the theme type:

Card Themes

Card themes (broadcast, street-sign, wood, do-not-disturb, marquee, dot-matrix) use CSS transform: scale() on an inner wrapper. When scale < 1, the card floats centered on a transparent background. Use the fill param to set the card face color, and bg for the page background.

Extended Themes

All other themes apply scale as a font-size multiplier: the auto-fit result is multiplied by the scale value. Background effects remain fullscreen.

Time Product

Clock themes support scale (0.1–3), position, and padding as layout parameters. When scale ≠ 1 or padding > 0, a wrapper is created inside the container with real dimensions (not CSS transform), so themes render at the correct size. Values < 1 shrink the clock; values > 1 enlarge it beyond the container, with overflow:hidden clipping the excess. Use position to control which portion of the enlarged clock is visible (e.g., top-left anchors the top-left corner). padding adds spacing from container edges.

led.run/time?t=digital&scale=0.5&position=top-left&padding=3
led.run/time?t=digital&scale=2&position=center            — 2× zoom, centered
led.run/time?t=digital&scale=2&position=top-left           — 2× zoom, top-left anchor

Casting

The Cast feature uses the W3C Presentation API to display your content on a secondary screen. Currently supported in Chrome and Edge.

  • Click the Cast button in the toolbar to select a display
  • The receiver shows a clean display without toolbar or controls
  • The connection persists across page navigations
The Cast button only appears in browsers that support the Presentation API.
