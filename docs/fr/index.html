Documentation led.run

Transformez n'importe quel écran en panneau texte, effet lumineux, visualiseur sonore, affichage horloge, code QR ou filtre caméra. Pas d'application, pas de compte — juste une URL.

Comment ça marche

led.run utilise des URL pour créer des affichages. Le chemin et la chaîne de requête configurent tout. Ouvrez l'URL dans n'importe quel navigateur et votre affichage est prêt — aucun traitement serveur, aucun compte nécessaire.

Panneaux texte

led.run/HELLO?theme=neon&color=ff00ff

Protocole URL

Six produits, six schémas d'URL :

Text:   led.run/HELLO?t=neon              Path = contenu texte
        led.run/text/HELLO?t=neon         Préfixe /text/ explicite
Light:  led.run/light?t=disco             Préfixe produit + paramètres
Sound:  led.run/sound?t=bars              Préfixe produit + paramètres
Time:   led.run/time?t=digital            Préfixe produit + paramètres
QR:     led.run/qr/HELLO?t=neon           Contenu dans le chemin + paramètres
Camera: led.run/camera?t=ascii            Préfixe produit + paramètres
  • Text — le chemin URL après / devient le texte affiché. Les espaces peuvent être + ou %20.
  • QR — le contenu à encoder suit le préfixe /qr/. Les URL dans le contenu doivent être encodées.
  • Light, Sound, Time & Camera — utilisent des préfixes de chemin réservés. Toute la configuration se fait via les paramètres de requête.
  • Tous les paramètres sont des « indications de préférence » optionnelles — les thèmes/effets/visualiseurs/horloges décident s'ils les utilisent.
Préfixes réservés : text, light, sound, time, qr, camera sont réservés. Pour afficher le mot « light » en tant que texte, utilisez led.run/text/light.

Texte — Aperçu

Le produit Texte transforme n'importe quel chemin URL en affichage plein écran. Il comprend 30 thèmes intégrés, chacun conçu pour une ambiance et un cas d'utilisation spécifiques. Les thèmes prennent en charge deux modes d'affichage : panneau statique et défilement.

led.run/HELLO                          Panneau simple
led.run/DO+NOT+DISTURB?t=do-not-disturb   Sélection de thème
led.run/OPEN?t=neon&c=ff2d78&bg=0a0008   Couleurs personnalisées
led.run/WELCOME?mode=flow&speed=120      Bandeau défilant

Modes d'affichage

Les thèmes texte prennent en charge deux modes d'affichage :

Mode panneau (Sign)

Affichage statique plein écran. Le texte est automatiquement ajusté pour remplir l'espace disponible. Idéal pour les messages courts comme « OPEN », « ON AIR » ou « DO NOT DISTURB ».

Mode défilement (Flow)

Bandeau défilant. Le texte défile continuellement à travers l'écran. Idéal pour les messages longs, les annonces ou les téléscripteurs.

Détection automatique : Si aucun paramètre mode n'est défini, la plupart des thèmes utilisent par défaut le mode sign pour les textes courts (≤10 caractères) et le mode flow pour les textes plus longs.

Paramètres texte

Ces paramètres sont reconnus par les thèmes texte. Chaque thème fournit ses propres valeurs par défaut.

Param Alias Type Défaut Description
theme t string default ID du thème (voir Thèmes)
mode — string auto sign (statique) ou flow (défilement)
color c hex par thème Couleur du texte (format couleur)
bg — hex par thème Couleur de fond
speed — number par thème Vitesse de défilement en px/s (mode flow)
direction dir string left Direction de défilement : left ou right
font — string par thème Nom de la famille de police
scale — number 1 Multiplicateur d'échelle (Text: 0.1–1, Time: 0.1–3). Voir Échelle
fill — hex par thème Fond de la face carte (thèmes carte, quand scale < 1)
wakelock w boolean true Maintenir l'écran allumé (empêche la mise en veille)
cursor cur number — Délai de masquage automatique du curseur en millisecondes
lang l string auto Langue de l'interface : en, zh, ja, ko, es, fr, de

Lumière — Aperçu

Le produit Lumière transforme n'importe quel écran en effet lumineux plein écran. Utilisez-le comme lampe torche, stroboscope, lumière d'ambiance ou affichage décoratif. Les 18 effets intégrés vont du pratique (solid, SOS) à l'atmosphérique (aurora waves, lava lamp).

led.run/light?t=solid              Lampe torche blanche
led.run/light?t=disco&speed=5      Lumières disco rapides
led.run/light?t=candle&warmth=9    Bougie extra chaude
led.run/light?t=aurora-waves       Bandes d'aurore lumineuses
Note : /light sans ?t= ouvre la page d'accueil. Les liens de partage incluent toujours ?t=.

QR — Aperçu

Le produit QR génère des codes QR plein écran à partir de tout texte, URL ou donnée. Il comprend 4 thèmes visuels. Le contenu à encoder est placé directement dans le chemin URL après /qr/.

led.run/qr/HELLO                       QR texte simple
led.run/qr/https%3A%2F%2Fexample.com   URL (encodée)
led.run/qr/HELLO?t=neon&ec=H           Thème néon, correction d'erreur élevée
led.run/qr/HELLO?t=dot&shape=diamond   Thème point avec modules losange
Note : Les URL dans le contenu QR doivent être URL-encodées (ex. https%3A%2F%2F pour https://) car ? serait interprété comme délimiteur de chaîne de requête. /qr sans contenu ouvre la page d'accueil.

Color Format

Colors are specified as hex values without the # prefix:

  • 6-digit RRGGBB — e.g. ff0000 for red
  • 8-digit AARRGGBB — e.g. 80ff0000 for 50% transparent red
led.run/HELLO?color=00ff41          Green text
led.run/HELLO?bg=1a0033&c=ff00ff   Purple bg, magenta text

Scale & Card Themes

The scale parameter controls display size. Its behavior depends on the theme type:

Card Themes

Card themes (broadcast, street-sign, wood, do-not-disturb, marquee, dot-matrix) use CSS transform: scale() on an inner wrapper. When scale < 1, the card floats centered on a transparent background. Use the fill param to set the card face color, and bg for the page background.

Extended Themes

All other themes apply scale as a font-size multiplier: the auto-fit result is multiplied by the scale value. Background effects remain fullscreen.

Time Product

Clock themes support scale (0.1–3), position, and padding as layout parameters. When scale ≠ 1 or padding > 0, a wrapper is created inside the container with real dimensions (not CSS transform), so themes render at the correct size. Values < 1 shrink the clock; values > 1 enlarge it beyond the container, with overflow:hidden clipping the excess. Use position to control which portion of the enlarged clock is visible (e.g., top-left anchors the top-left corner). padding adds spacing from container edges.

led.run/time?t=digital&scale=0.5&position=top-left&padding=3
led.run/time?t=digital&scale=2&position=center            — 2× zoom, centered
led.run/time?t=digital&scale=2&position=top-left           — 2× zoom, top-left anchor

QR Product

QR themes support the same wrapper mode as Time: scale (0.1–3), position, padding, and fill. When scale ≠ 1 or padding > 0, a wrapper is created. fill controls the outer container background when the wrapper is active.

led.run/qr/HELLO?scale=0.5&position=center&fill=1a1a2e
led.run/qr/HELLO?scale=2&position=top-left

Camera Product

Camera effects support scale (0.1–3) and position for a picture-in-picture effect when scale < 1. No padding or fill params — simplified version of the wrapper mode.

led.run/camera?t=ascii&scale=0.5&position=bottom-right

Casting

The Cast feature uses the W3C Presentation API to display your content on a secondary screen. Currently supported in Chrome and Edge.

  • Click the Cast button in the toolbar to select a display
  • The receiver shows a clean display without toolbar or controls
  • The connection persists across page navigations
The Cast button only appears in browsers that support the Presentation API.
